    # 类似unocss的原子化css

    Status: Done Text: github有代码；简单版本

    # 基础：less

    • less和scss虽然写法不同但是基本思想一致；
    • 用less一些基本语法去定制化；→定制属于项目自己的原子化；

    # 如何写？

    • 最好去官网看语法 → https://lesscss.org/features/#extend-feature-extend-syntax (opens new window)
    • 先构思自己样式分类；整个项目大致所需；
    • question：当我使用mixin时候没法在vue文件中生效；→因为我没在vite.config.ts中引入全局
    export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': '/src'
    }
  },
  css: {
    preprocessorOptions:{
      less:{
        javascriptEnabled: true,
        **additionalData: "@import 'src/styles/index.less';"**
      }
    }
  }
})

    • unc.less

      -> unc.less

@import "color.less";
@margin-sizes: 5,10,16,20;
@padding-sizes:2,5,10,15,20;
// @height-sizes: 20,50,100,150,200;
@border-radius: 5,10,15;
@direction: {
    t: top;
    r: right;
    b: bottom;
    l: left;
  };
// 主基调颜色
.color {
    each(@set,{
     &-@{key} {
         color: @value;
     }
    })
 }
// 主基调背景颜色
 .background{
    each(@set,{
        &-@{key} {
            background-color: @value;
        }
       })
 }
//  主基调布局
//eg:lay-f: flex布局; .f-c: 列不换行flex布局; .fj-s:主轴左对齐～; .fa-e: 交叉轴终点对齐～；
@layout:{
    b: block;
    ib: inline-block;
    f: flex;
    g: grid;
}
@flex-flow: {
    r: row nowrap;
    c: column nowrap;
    rl: row-reverse nowrap;
    cl: column-reverse nowrap;
    rn: row wrap;
    cn: column wrap;
    rln: row-reverse wrap;
    cln: column-reverse wrap;

};
@justify: {
    s: flex-start;
    e: flex-end;
    c: center;
    b: space-between;
    a: space-around;
}
@align: {
    s: flex-start,;
    e: flex-end;
    c: center;
    b: baseline;
    st: stretch;
};

each(@layout,{
    .lay{
      &-@{key} {
        display: @value;
    };
    }
});
each(@flex-flow,{
    .f{
        &-@{key}{
            display: flex;
            flex-flow: @value;
          }
    }
});
each(@justify,{
       .fj{
        &-@{key}{
            display: flex;
                justify-content: @value;
              }
       }
});
each(@align,{
           .fa{
            -@{key}{
                display: flex;
                    justify-content: @value;
                  }
           }
});
.flex-1{
    flex: 1;
}
//主基调高度height:10的倍数，max100px;
//eg: h-10=>height:100px;
each(range(10),{
    .h-@{value} {
        height:(@value*10px);
    }
})
//主基调高度width:10的倍数，max100px;
//eg: w-10=>width:100px;
each(range(10),{
    .w-@{value} {
        width:(@value*10px);
    }
})
// 主基调margin
each(@margin-sizes, {
  .m {
    &-@{value}{
      margin: 1px * @value;
    }
    // each 函数默认使用 @value,@key,@index 为变量名
    // 当通过 #() 或者 .() 的样式为开头传递变量名时
    // each 函数会获取传入的变量的名并按顺序将其绑定为 @value,@key,@index 的值
    each(@direction, .(@v, @k) {
      &@{k}-@{value} {
        margin-@{v}: 1px *  @value;
      }
    });
  }
});

//主基调padding
each(@padding-sizes,{
    .p {
        &-@{value}{
            padding:1px * @value;
        }
        each(@direction , .(@v, @k){
            &@{k}-@{value} {
                padding-@{v}: 1px * @value;
            }
        })
    }
});

//主基调悬浮
.hover {
    &:hover {
        cursor: pointer;
        background-color: var(--color-hover);
    }
}
//聚焦
.focus {
    &:focus {
        background-color: var(--color-test);
    }
}
//主基调圆角
each(@border-radius,{
    .br {
        &-@{value}{
            border-radius:1px * @value;
        }
    }
})

    # 我当时总结的速记

    原因：这种不易形成项目式使用；更偏向开放式；我们需要规范使用情况；所以需要基础固定

    • *自定义一套属于自己的*unocss->因为是简洁项目，所需样式基本一致，没有太多复杂样式>这样大大减少复写，以及成员自由发挥样式
    • 颜色：定义网页元素的颜色，如字体颜色、背景色等。
    • 字体：定义字体样式，如字体类型、大小、加粗、斜体等。
    • 布局：定义页面元素的布局，如宽度、高度、边距、内边距等。
    • 背景：定义页面元素的背景，如背景图片、背景颜色等。
    • 边框：定义页面元素的边框，如边框宽度、边框颜色、边框样式等。
    • 盒模型：定义页面元素的盒模型，如盒模型的宽度、高度、边距、内边距等
    • 定位：定义页面元素的定位，如定位方式、定位元素等。
    • 动画：定义页面元素的动画效果，如淡入淡出、旋转、缩放等
